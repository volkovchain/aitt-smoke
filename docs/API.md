# aitt-smoke — API и внутренние компоненты

Документ описывает HTTP-эндпоинты, экспонируемые сервисом, и функции
пакета `main`, реализующие его поведение.

Все утверждения проверены по исходному коду в `main.go` и `main_test.go`.

---

## 1. HTTP API

Сервис слушает один TCP-адрес и обслуживает все запросы через единственный
`*http.ServeMux`, собранный функцией [`newMux`](#newmux---httpservemux).

Маршрутизация выполняется через `http.ServeMux`, поэтому:

* пути матчатся точно (`/health`, `/version`);
* для незарегистрированного пути возвращается `404 Not Found` (поведение
  стандартного `ServeMux`);
* HTTP-метод не проверяется — `HEAD`, `POST` и т.п. также попадут в
  обработчик и получат то же тело.

### 1.1 `GET /health`

Проверка живости сервиса.

| Свойство          | Значение                                       |
|-------------------|------------------------------------------------|
| Метод             | любой (рекомендуется `GET`)                    |
| Путь              | `/health`                                      |
| Статус            | `200 OK` (выставляется неявно при первой записи)|
| Заголовок         | `content-type: application/json`               |
| Тело              | `{"ok":true}\n` (с завершающим переводом строки)|

Пример:

```bash
$ curl -i http://127.0.0.1:8081/health
HTTP/1.1 200 OK
content-type: application/json
Date: ...
Content-Length: 12

{"ok":true}
```

Замечание: ключ заголовка в исходниках записан в нижнем регистре
(`"content-type"`); `net/http` нормализует его в `Content-Type` при
сериализации ответа, но при анализе сырого кода это полезно знать.

### 1.2 `GET /version`

Возвращает строковую версию контракта сервиса.

| Свойство  | Значение                          |
|-----------|-----------------------------------|
| Метод     | любой (рекомендуется `GET`)       |
| Путь      | `/version`                        |
| Статус    | `200 OK` (выставляется явно через `WriteHeader`) |
| Заголовок | `Content-Type: text/plain`        |
| Тело      | `v1` (ровно 2 байта, без `\n`)    |

Пример:

```bash
$ curl -i http://127.0.0.1:8081/version
HTTP/1.1 200 OK
Content-Type: text/plain
Date: ...
Content-Length: 2

v1
```

## 2. Пакет `main`

Сервис представляет собой один Go-файл (`main.go`) в пакете `main`.
Экспортируемых символов нет; обе функции — `newMux` и `main` — внутренние,
но описаны здесь для полноты.

### `newMux() *http.ServeMux`

Создаёт и возвращает `*http.ServeMux` с зарегистрированными обработчиками
`/health` и `/version` (поведение каждого см. в разделе [HTTP API](#1-http-api)).

* **Параметры:** нет.
* **Возвращает:** не-nil `*http.ServeMux`, готовый к использованию.
* **Ошибки/паники:** не возвращает ошибок, не паникует.
* **Назначение:** изолировать сборку роутера, чтобы его можно было
  переиспользовать в тестах без поднятия `http.ListenAndServe`.

Пример (из `main_test.go`):

```go
mux := newMux()
srv := httptest.NewServer(mux)
defer srv.Close()

resp, err := http.Get(srv.URL + "/version")
if err != nil {
    t.Fatalf("GET /version: %v", err)
}
defer resp.Body.Close()
// resp.StatusCode == 200, body == "v1"
```

### `main()`

Точка входа исполняемого файла.

Алгоритм:

1. `mux := newMux()`.
2. `addr := os.Getenv("LISTEN_ADDR")`; если пусто — `addr = ":8081"`.
3. Печатает `listening on <addr>` в stdout через `fmt.Println`.
4. Вызывает `http.ListenAndServe(addr, mux)`.
5. Любая возвращённая ошибка пишется в stderr, после чего вызывается
   `os.Exit(1)`.

* **Параметры/возврат:** соответствуют сигнатуре `func main()`.
* **Исключения/ошибки:** ошибки `ListenAndServe` (например, занятый порт,
  невалидный адрес) приводят к завершению процесса кодом `1` с сообщением
  в stderr.
* **Особенности:** graceful shutdown не реализован — процесс
  останавливается сигналом (`SIGINT`/`SIGTERM`).

Пример запуска:

```bash
# адрес по умолчанию
$ go run .
listening on :8081

# нестандартный адрес
$ LISTEN_ADDR=127.0.0.1:9000 go run .
listening on 127.0.0.1:9000
```

## 3. Тесты

Файл `main_test.go` содержит один тест:

### `TestVersionRoute(t *testing.T)`

Проверяет контракт эндпоинта `/version`:

1. Собирает роутер через `newMux()`.
2. Поднимает `httptest.NewServer(mux)` (получает локальный URL со
   случайным портом).
3. Делает `GET <srv.URL>/version`.
4. Сверяет:
   * `resp.StatusCode == 200`;
   * заголовок `Content-Type` содержит подстроку `text/plain`;
   * тело ответа равно `"v1"`.

Тест не использует моки и не обращается к сети за пределы loopback;
его можно запускать в любой изолированной среде, где доступен Go-toolchain.

Запуск:

```bash
go test ./...
go test -v -race ./...
```
