# aitt-smoke

Минимальный HTTP-сервис на Go, используемый как smoke-тест для оркестратора
`aitt`. Сервис поднимает два эндпоинта на одном порту и предназначен для того,
чтобы убедиться, что окружение оркестратора способно:

* собрать и запустить произвольное Go-приложение,
* достучаться до его сетевых эндпоинтов,
* получить ожидаемые ответы (`/health`, `/version`).

Сервис намеренно не содержит бизнес-логики — это «заглушка», по которой
проверяется работоспособность инфраструктуры вокруг.

## HTTP API

| Метод | Путь       | Content-Type                | Ответ          |
|-------|------------|-----------------------------|----------------|
| GET   | `/health`  | `application/json`          | `{"ok":true}\n`|
| GET   | `/version` | `text/plain`                | `v1`           |

Подробное описание поведения каждого эндпоинта приведено в
[`docs/API.md`](docs/API.md).

## Требования к окружению

* **Go** версии **1.22** или новее (значение `go` директивы в `go.mod`).
* Утилита `make` — опционально, для целей в `Makefile`.
* `curl` — опционально, используется целью `make health`.
* Прямой доступ к TCP-порту, который слушает сервис (по умолчанию `:8081`).

Внешних зависимостей у проекта нет — `go.mod` не содержит блока `require`,
используется только стандартная библиотека (`net/http`, `io`, `fmt`, `os`).

## Установка и настройка

```bash
git clone https://github.com/volkovchain/aitt-smoke.git
cd aitt-smoke
go build ./...
```

### Переменные окружения

| Имя           | Назначение                                 | По умолчанию |
|---------------|--------------------------------------------|--------------|
| `LISTEN_ADDR` | Адрес, на котором сервис принимает запросы | `:8081`      |

Значение `LISTEN_ADDR` передаётся напрямую в `http.ListenAndServe` и
поддерживает любой формат, принимаемый стандартной библиотекой Go
(например, `:8080`, `127.0.0.1:9000`, `0.0.0.0:8081`).

## Локальный запуск

Самый простой способ — через `make`:

```bash
make run
```

Эквивалентный вызов без `make`:

```bash
go run .
```

После запуска сервис печатает в stdout строку вида:

```
listening on :8081
```

Проверка живости:

```bash
make health
# или вручную
curl -fsS http://127.0.0.1:8081/health
# {"ok":true}

curl -fsS http://127.0.0.1:8081/version
# v1
```

Запуск на нестандартном адресе:

```bash
LISTEN_ADDR=127.0.0.1:9000 go run .
```

Остановка — `Ctrl+C` (сервис не реализует graceful shutdown, прерывание
завершает процесс).

## Запуск тестов

В репозитории есть один тест — `TestVersionRoute` в `main_test.go`, который
поднимает `newMux()` через `httptest.NewServer` и проверяет статус,
`Content-Type` и тело ответа `/version`.

```bash
go test ./...
```

С подробным выводом и проверкой race-условий:

```bash
go test -v -race ./...
```
